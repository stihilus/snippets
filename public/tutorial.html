The Mysterious 's' Object: Your Magic Wand

Now, let's talk about our little friend 's'. Imagine you're at a fancy cocktail party (the kind where code is the drink of choice). You want to use p5.js functions, but you don't want to be that person who hogs all the global variables. Enter 's' – your personal butler who fetches all the p5.js goodies for you.

Why 's'? Well, 's' stands for "sketch," but we like to think it's short for "super-cool-code-savior." By using 's.', you're basically saying, "Hey, p5.js, could you please pass me that function? Thanks, you're a gem!" It keeps your code tidy and plays nice with others in the grand ballroom of your web page. 

    // Instead of crashing the party with:
    function setup() {
      createCanvas(windowWidth, windowHeight);
    }

    // Be a polite guest and use:
    s.setup = () => {
      s.createCanvas(s.windowWidth, s.windowHeight);
    }

    // It's not just good manners, it's good coding!

Compressing Code: The Art of Digital Origami

Ready to fold your code into a tiny, beautiful shape? Here are some ninja techniques:

  • Use short variable names (x instead of positionX)
  • Embrace the 's' object like it's your coding BFF
  • Squish multiple statements into one line (semicolons are your friends)
  • Arrow functions: Because who needs the word "function" anyway?
  • Implicit returns in arrow functions (return? What return?)
  • Shorthand operators (+=, *=) – because typing is so last year
